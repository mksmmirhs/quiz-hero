1.What is the difference between map, filter, forEach, find method?

Ans: All four method works on array but the difference is given bellow

  • map: it loops through each element of the array and modifies each element according to the logic given on callback function and returns the result as an array.
  • filter: it loops through all the element of an array and return elements as an array by checking the condition pass through callback function.
  • forEach: it loops through each element of an array according to the callback function but do not returns the result
  • find: find method loops though array and returns the first element matching the condition of callback function.

2.What is the difference between localStorage, sessionStorage, cookies?

  • localStorage: localStorage stores the data in the browser but it preserves the data even after browser close or session end.
  • sessionStorage: sessionStorage stores the data available to a particular session. this means data vanishes after browser window close or session end.
  • cookies: this is small file saved in users devise by websites. it preserves the file even after browser close. this data can be access by server and the user sets the cookies expiration date.

3. How does the event loop works?

Ans: Javascript is a single threaded language. To run asynchronous task whether not blocking the execution event loop kicks in. When asynchronous tasks comes it removes from call stack to event loop removes it from call stack to callback queue. then it waits to finish the asynchronous task. then the task is finish event loop check if the call stack is free or not. if the call stack is free event loop removes the task from queue and push it to call stack. This is how javascript deals with asynchronous task in a single thread non blocking way.
